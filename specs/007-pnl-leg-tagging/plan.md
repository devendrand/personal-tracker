# Implementation Plan: PnL Leg Tagging & Strategy Groups

**Branch**: `007-pnl-leg-tagging` | **Date**: 2026-02-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-pnl-leg-tagging/spec.md`

---

## Summary

Replace the existing `strategy_type` field on transactions with a simplified `leg_type` enum (CSP | CC | BUY | SELL). Add a new `StrategyGroup` entity that lets users manually group related legs into named strategy cycles (e.g., "AAPL Wheel Q1 2025"). Expose a PnL view API and UI that aggregates realized PnL as a Ticker → StrategyGroup → Legs hierarchy. PnL per leg equals the transaction's `amount` field directly (already net cash in/out).

## Implementation Phases

### Phase 0: Database & Models

**Goal**: Establish the new data model before any API or UI work.

1. **Edit `api/app/models/trade.py`**:
   - Remove `StrategyType` enum (6 values)
   - Add `LegType(StrEnum)` with values: `CSP`, `CC`, `BUY`, `SELL`
   - Add `StrategyGroup(Base)` model (see data-model.md)
   - Update `Transaction`: remove `strategy_type`, add `leg_type` and `strategy_group_id` FK

2. **Generate and run Alembic migration**:
   ```bash
   cd api && uv run alembic revision --autogenerate -m "add_leg_type_and_strategy_group"
   uv run alembic upgrade head
   ```
   Migration ops:
   - `op.drop_column("transaction", "strategy_type")`
   - `op.add_column("transaction", Column("leg_type", String(10), nullable=True))`
   - `op.create_table("strategy_group", ...)` with UUID pk, user_sub index, name, symbol, timestamps
   - `op.add_column("transaction", Column("strategy_group_id", String(36), ForeignKey("strategy_group.id"), nullable=True))`
   - Add index: `ix_transaction_strategy_group_id`

### Phase 1: Backend — Schemas, Services, Routers

**Order matters: schemas → service → routers (tests first for each).**

1. **`api/app/schemas/trade.py`** — Update:
   - Replace `StrategyType` import with `LegType`
   - Update `TransactionResponse`: `leg_type: LegType | None`, add `strategy_group_id: str | None`
   - Replace `TransactionStrategyTypePatchRequest` → `LegTypePatchRequest(leg_type: LegType | None)`
   - Replace `StrategyTypeOption` → `LegTypeOption(value: LegType, label: str, description: str)`

2. **`api/app/schemas/strategy_group.py`** — New:
   - `StrategyGroupCreate(name: str, symbol: str)`
   - `StrategyGroupResponse(id, name, symbol, created_at)`
   - `AssignLegRequest(strategy_group_id: str | None)`

3. **`api/app/schemas/pnl.py`** — New:
   - `LegPnL`, `StrategyGroupPnL`, `TickerPnL`, `PnLSummaryResponse` (see data-model.md)

4. **`api/app/services/pnl_service.py`** — New:
   - `calculate_pnl(transactions: list[Transaction], groups: dict[str, StrategyGroup]) -> PnLSummaryResponse`
   - Algorithm: group by symbol → group by strategy_group_id (None = "Ungrouped") → `realized_pnl = amount`

5. **`api/app/routers/transactions.py`** — Update:
   - Rename `GET /transactions/strategy-types` → `GET /transactions/leg-types`
   - Rename `PATCH /transactions/{id}/strategy-type` → `PATCH /transactions/{id}/leg-type`
   - Add `PATCH /transactions/{id}/strategy-group` → assign/unassign leg from group (validates symbol match)
   - Update `GET /transactions` filter: `leg_type` query param instead of `strategy_type`

6. **`api/app/routers/strategy_groups.py`** — New:
   - `GET /strategy-groups` → `list[StrategyGroupResponse]`
   - `POST /strategy-groups` → `StrategyGroupResponse` (201)
   - `DELETE /strategy-groups/{id}` → 204, nullifies `strategy_group_id` on all linked legs

7. **`api/app/routers/pnl.py`** — New:
   - `GET /pnl` → `PnLSummaryResponse`
   - Queries tagged transactions + their groups for `current_user`, calls `pnl_service.calculate_pnl()`

8. **`api/app/routers/__init__.py`** + **`api/app/main.py`** — Export and mount new routers.

### Phase 2: Frontend

**Order: models → api-client → transactions update → strategy-groups feature → pnl feature → routing.**

1. **`web/src/app/features/transactions/`** — Update:
   - Replace `strategy_type` dropdown with `leg_type` dropdown (CSP, CC, BUY, SELL)
   - Call `GET /api/transactions/leg-types` for options
   - Call `PATCH /api/transactions/{id}/leg-type` on change
   - Add "Assign to Group" action per row → modal/select → calls `PATCH /api/transactions/{id}/strategy-group`

2. **`web/src/app/features/strategy-groups/`** — New:
   - `strategy-groups.component.ts`: list groups, create form (name + symbol), delete button
   - `strategy-groups.routes.ts`

3. **`web/src/app/features/pnl/`** — New:
   - `pnl.models.ts`: `LegPnL`, `StrategyGroupPnL`, `TickerPnL`, `PnLSummaryResponse`
   - `pnl.component.ts`: fetches `GET /api/pnl`, renders collapsible tree with signals
   - `pnl.routes.ts`

4. **`web/src/app/core/api-client.service.ts`** — Add:
   - `getLegTypes()`, `patchLegType(id, legType)`, `patchStrategyGroup(id, groupId | null)`
   - `getStrategyGroups()`, `createStrategyGroup(name, symbol)`, `deleteStrategyGroup(id)`
   - `getPnL()`

5. **`web/src/app/app.routes.ts`** — Add lazy routes for `pnl` and `strategy-groups` with `authGuard`.

6. **Nav component** — Add links for "PnL" and "Strategy Groups".

# Implementation Plan: PnL Commission Fix & Additional Metrics

**Branch**: `008-fix-pnl-commission` | **Date**: 2026-02-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/008-fix-pnl-commission/spec.md`

---

## Summary

Fix a data-correctness bug where `realized_pnl` per leg uses the gross `amount` field instead of `amount + commission` (commission is stored as a negative value from E*TRADE CSV). Also add two new PnL metrics — `transaction_count` and `total_commission` — at all aggregation levels (group, ticker, grand total), surfaced as new columns on the existing PnL page.

**No database migration required** — the `commission` column already exists on `transaction`.

## Implementation Phases

### Phase 1: Backend Schema + Service Fix (P1 — data correctness)

**Goal**: Fix `realized_pnl` and expose commission metrics in the API.

**Step 1.1 — Write failing tests first (TDD)**

Add to `api/tests/test_pnl.py`:

- `test_pnl_commission_deducted` — CSV with `-0.65` commission; assert `realized_pnl = 149.35` and `commission = "0.65"`
- `test_pnl_null_commission_treated_as_zero` — CSV with null commission; assert `realized_pnl = amount`, `commission = "0.00"`
- `test_pnl_group_transaction_count` — group with 3 legs; assert `transaction_count = 3`
- `test_pnl_total_commission_aggregation` — legs with `-0.65`, `-0.65`, `0.00`; assert `total_commission = "1.30"` at group
- `test_pnl_ticker_rollup` — two groups each with commission; assert ticker-level `transaction_count` and `total_commission` are summed correctly
- `test_pnl_grand_total_fields` — assert `total_transaction_count` and `total_commission` on `PnLSummaryResponse`
- `test_pnl_commission_displayed_as_positive` — stored `-0.65` → returned `"0.65"` (not `"-0.65"`)

Also add assertions for `transaction_count`, `total_commission`, `total_transaction_count` to **all existing tests** (most will assert `0` since existing fixtures have `commission=0.00`).

**Step 1.2 — Update Pydantic schemas**

In `api/app/schemas/pnl.py`:
- `LegPnL`: add `commission: Decimal`
- `StrategyGroupPnL`: add `transaction_count: int`, `total_commission: Decimal`
- `TickerPnL`: add `transaction_count: int`, `total_commission: Decimal`
- `PnLSummaryResponse`: add `total_transaction_count: int`, `total_commission: Decimal`

**Step 1.3 — Fix service logic**

In `api/app/services/pnl_service.py`:

1. Fix per-leg computation:
   - `commission_raw = Decimal(str(txn.commission)) if txn.commission is not None else Decimal("0")`
   - `realized_pnl = amount + commission_raw`
   - `commission = abs(commission_raw)`
   - Pass `commission=commission` to `LegPnL(...)`

2. Add group-level metrics after building `leg_pnls`:
   - `transaction_count = len(leg_pnls)`
   - `total_commission = sum((leg.commission for leg in leg_pnls), Decimal("0"))`

3. Add ticker-level metrics after building `group_results`:
   - `transaction_count = sum(g.transaction_count for g in group_results)`
   - `total_commission = sum((g.total_commission for g in group_results), Decimal("0"))`

4. Add grand-total metrics after building `ticker_results`:
   - `total_transaction_count = sum(t.transaction_count for t in ticker_results)`
   - `total_commission = sum((t.total_commission for t in ticker_results), Decimal("0"))`

5. Update the empty-state early return:
   ```python
   return PnLSummaryResponse(
       total_realized_pnl=Decimal("0"),
       total_transaction_count=0,
       total_commission=Decimal("0"),
       tickers=[],
   )
   ```

**Step 1.4 — Run backend tests**

```bash
cd api && uv run pytest tests/test_pnl.py -v
```

All tests must pass (including existing ones whose expected PnL values are unchanged since `commission=0.00` in their fixtures).
